【地圖資訊系統/位置分派（google map介接）】實作規格：
1. 前台地圖資訊模組須整合 Google Map 服務，於案件通報、案件分派過程中，將案件地點資訊以圖資定位方式於地圖上即時標示與顯示，並可檢視案件分布狀況 (出處: 勤務及案管整合智慧系統需求說明書 P.24),2. 管理端應能以地圖方式快速查閱全部或條件篩選的案件分佈，包含查詢特定案件類別、時間區間、區域及案件狀態之案件，並於地圖上一目了然顯示 (出處: 勤務及案管整合智慧系統需求說明書 P.25),3. 案件收件時須自動將地址或坐標資訊完成地理編碼，並與 Google Map API 進行串接，將正確地理位置定位於地圖标注 (出處: 勤務及案管整合智慧系統需求說明書 P.24),4. 出勤及分派功能於地圖上可點選案件或區塊匯聚進行任務指派，並顯示待派案件、已分派案件與已結案案件不同顏色標記，提升分派效率 (出處: 勤務及案管整合智慧系統需求說明書 P.25),5. 各案件點選後可彈窗顯示該案件詳細資訊（如案件編號、地址、案件類型、案件狀態、報案人資料、省略個資顯示）、現場照片，地圖可協助路線規劃 (出處: 勤務及案管整合智慧系統需求說明書 P.24),6. 案件通報介面須於通報時即時於地圖上標示輸入（或自動偵測）之地點，供通報人及承辦人即時檢查其正確性，並可支援地點拖曳修正 (出處: 勤務及案管整合智慧系統需求說明書 P.24),7. 地圖資訊模組須符合行動裝置操作（支援 RWD 響應式設計），使承辦人員於外勤時可透過手機/平板查閱案件地圖 (出處: 勤務及案管整合智慧系統需求說明書 P.25),8. 系統須考量 Google Map 介接金鑰及流量管理、API 錯誤處理及相關日誌紀錄等維運防呆機制 (建議實作)
